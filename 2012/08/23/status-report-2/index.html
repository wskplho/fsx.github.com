Status Report #2

23 Aug 2012

Time for another status report. I didn't have a lot of time for my projects lately and nothing significant happened. So it's time to set priorities, get things done and put less important things on hold.

Momoko

Version 0.5.0 has been release last month and since then I've been rewriting the code. Mostly removing unnecessary layers and restructuring everything to fit in new functionality. Mostly stuff provided by Psycopg2 that is not supported in asynchronous mode and asynchronous notifications.

The code can be found in the rewrite branch on Github. Everything that's on to do list is put into todo.rst with a description of how I think it can be implemented.

My internship will start in three weeks and Momoko will most likely be used in the web application that is going to be made. More time for Momoko. :D

Misaka

It's been a while since the last release and some bugs popped up that need to be fixed. Misaka doesn't seem to compile on Windows with Visual Studio. And the last time I tried compiling was last year with MinGW, but that's not the default compiler configured in Python since I had to change a configuration file somewhere.

I have access to a Windows 7 machine and maybe Windows 8 in some months. It may be it's also interesting to do continuous integration on Windows, but Travis CI doesn't support Windows and Shining Panda doesn't include Windows in it's FOSS plan.

Besides fixing Windows support I also plan to replace the Cython parts with pure C code. This removes the extra compile step needed when developing and unreadable C code generated by Cython. And bugs can also be less confusing. Like the bug that only appeared in Python 3 when the lines of code for utf-8 to bytes conversion were in a certain order...

Akane

Akane is still a work in progress. Many things will be tried to improve the performance. Like implementing various things in C. For example, the replies are now parsed using hiredis-py, but that will be replaced by my own reply parser implemented in C.

Commands are very simple to add, but extended features are not. I haven't even thought about it yet. It's not a high priority project, but just send me an email if you need anything.

Documentation

A simple to use documentation generator will be made to replace the documentation of my own projects. I just don't like reStructuredText and Sphinx. Don't hate me.

The things I have in mind are Markdown for markup, generated API pages for Python, manually written API pages, multi-language support and a built-in webserver and a file watcher for a quick preview when writing documentation.

I have some more things in mind, but the above are the features I really need. Except multi-language support, but implementing those kind of things is not easy in the later stages of development.

FluxBB/php-utf8/Mitto

I've stopped using PHP and because of that I didn't contribute anything to FluxBB for a long time. So now officially stopped being a FluxBB developer. This means I also stop maintaining php-utf8.

Mitto is put on hold, because I don't use Thrift and requires time and dedication to continue with the project. Which I currently lack, because I want work on Misaka and Momoko.
